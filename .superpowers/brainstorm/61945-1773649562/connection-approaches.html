Wie verbinden wir die 4 Schritte?

Aktuell sind die Tabs isoliert — der rote Faden von der Kurve zum Bitcoin-Schlüssel fehlt. Hier sind 3 Ansätze:

A

Pipeline-Bar + Kontext-Brücken

Ein visueller Fortschrittsbalken über den Tabs zeigt den Weg: Kurve → Addition → Skalar-Mul → Schlüssel. Zwischen den Tabs erscheint ein kurzer "Brückentext", der erklärt, warum der nächste Schritt nötig ist.

1 Die Kurve → 2 Addition → 3 Skalar-Mul → 4 Schlüssel

"Du kennst jetzt die Kurve. Aber was kann man mit Punkten darauf tun? → Punkt-Addition"

Pro

  • Minimaler Eingriff — Tabs bleiben wie sie sind
  • User sieht jederzeit wo er im Gesamtbild steht
  • Brückentexte erklären das "Warum"

Contra

  • Narrative Verbindung bleibt rein textuell
  • Tabs teilen immer noch keinen State
B

Pipeline-Bar + Live-Datenfluss

Wie A, aber zusätzlich fließen Ergebnisse zwischen Tabs weiter: Der Punkt den du in Tab 2 addierst, wird in Tab 3 als Basis für die Skalarmultiplikation verwendet. Das Ergebnis von Tab 3 erscheint in Tab 4 als Public Key.

Tab 2: P + Q = R ──▶ R wird Basispunkt in Tab 3
Tab 3: n × G = P ──▶ P wird Public Key in Tab 4

Pro

  • User erlebt den Zusammenhang statt ihn nur zu lesen
  • Jeder Tab baut auf dem vorherigen auf
  • "Aha-Moment" wenn der eigene Punkt zum Schlüssel wird

Contra

  • Komplexer: Shared State zwischen Tabs nötig
  • Tabs müssen auch standalone funktionieren
  • Mehr Implementierungsaufwand
C

Zusammenfassungs-Tab "Der Weg zum Schlüssel"

Die 4 bestehenden Tabs bleiben unverändert. Ein neuer 5. Tab (oder eine Intro-Sektion über den Tabs) zeigt den gesamten Weg kompakt auf einer Seite: Von der Kurve über Addition und Skalarmultiplikation bis zum Schlüssel — als animierte Pipeline/Infografik.

Pro

  • Bestehende Tabs bleiben komplett unverändert
  • Übersicht auf einen Blick
  • Klick auf Pipeline-Schritt öffnet den zugehörigen Tab

Contra

  • Redundanz: Zusammenfassung wiederholt Inhalte
  • Kein echtes "Erleben" der Verbindung

Meine Empfehlung: B

Ansatz B erzeugt den stärksten Lerneffekt — der User sieht seinen eigenen Punkt durch die Pipeline wandern und versteht dadurch intuitiv, wie Punkt-Addition → Skalarmultiplikation → Schlüssel zusammenhängen. Die Pipeline-Bar (aus A) ist dabei inklusive.
