Design-Zusammenfassung: "Der Weg zum Schlüssel"

Alle Entscheidungen auf einen Blick — bitte prüfen und bestätigen.

1. Pipeline-Bar (ersetzt aktuelle Tabs)

✓ Die Kurve
y² = x³ + 7
✓ Addition
P + Q = R
③ Skalar-Mul
n × G
🔒 Schlüssel
k → K

Verbundene Karten mit Mathe-Operationen. Abgeschlossene = Cyan, aktiv = Violett + Glow, gesperrt = ausgegraut mit Schloss.

2. Lineares Freischalten (Vertiefte Interaktion)

Tab 1 → 2 ≥ 2 Punkte auf die Kurve setzen + Infotext scrollen
Tab 2 → 3 Konstruktion (P+Q) abspielen + P oder Q einmal draggen
Tab 3 → 4 Slider bewegen + Schritt-für-Schritt-Modus einmal starten

Beim Erfüllen der Kriterien: animierter "Weiter"-Impuls in der Pipeline-Bar (nächste Karte leuchtet kurz auf).

3. Hybrider Datenfluss

Tab 1 → 2 │ Gesetzte Punkte erscheinen als mögliche P/Q-Startpositionen in Tab 2
Tab 2 → 3 │ Ergebnis R = P+Q wird als Basispunkt G in Tab 3 voreingestellt
Tab 3 → 4 │ Weltenwechsel: Skalar n wird zum Private Key, Canvas-Animation zeigt Übergang ℝ → 𝔽ₚ

4. Weltenwechsel-Animation (Tab 3 → 4)

GSAP-Timeline: Kurve fragmentiert → Punkte-Raster → Skalar animiert ins Private-Key-Feld → echte secp256k1-Berechnung startet.

5. Architektur: Shared State

EllipticCurvesVisualizer (State-Owner)
├─ placedPoints[] → Tab 1 setzt, Tab 2 liest
├─ additionResult → Tab 2 setzt, Tab 3 liest (Basispunkt)
├─ scalar → Tab 3 setzt, Tab 4 liest (Private Key)
├─ unlockedTabs → Set<1|2|3|4>
└─ completionCriteria → pro Tab tracken

State wird im übergeordneten EllipticCurvesVisualizer gehalten und per Props an die Tabs weitergegeben. Kein Context/Store nötig.
